Alibar

介绍

  • 页头窄条通栏，包括用户登录状态和网站导航，称为alibar

版本

  • @create terence.wangt 2013.10.10 Fdev5 版
  • @update terence.wangt 2014.01.21 增加黑色背景版

使用说明

  • alibar目前由前端通用负责维护，且只有一套版本
  • 目前全站alibar标准请移步 全站alibar规范
  • alibar对应的CSS及JS文件需要产品自己引入：
    灰色背景CSS文件： 
    <link href="http://style.c.aliimg.com/sys/css/universal/alibar/standard-v4-min.css" rel="stylesheet"/>
    黑色背景CSS文件： 
    <link href="http://style.c.aliimg.com/sys/css/universal/alibar/standard-v4-black.css" rel="stylesheet"/>

    JS文件：
    基于FDev5前端框架的页面：
    需要在模块的 AMD 依赖列表中引入 alibar id 即可： 
    define(['sys/alibar/1.0'], function(Alibar){
  $(function(){
	...
  })
});

    基于FDev4前端框架的页面：
    需要手工引入standard-v4-min.js文件 
    <script src="http://style.c.aliimg.com/sys/js/universal/alibar/standard-v4-min.js"></script>
  • 调用方式及合理位置如下： 
    <head>
    ...
    <link href="http://style.c.aliimg.com/sys/css/universal/alibar/standard-v4-min.css" rel="stylesheet"/>
    ...
</head>
<body>
    <div id="doc">
        <!--本行为CMS调用代码-->$!sandBoxRender.bandingTemplate("110350_110350_alibar_standard")
        <!--本行为外部应用调用代码-->$cmsTemplate.setTemplate("alibar/standard.vm")
        ...
        <script src="对应FDev框架版本的JS地址"></script>
    </div>
</body>

Methods

  • Alibar.refresh : 刷新用户状态
  • Alibar.purchaselistRefresh : 刷新进货单信息

Event

  • delitem.alibar : 用户删除进货单数据时候，在document上分发事件

个性化

  • 目前alibar允许个性化，但只限于:底色、文字颜色、下拉菜单隐藏\显示图标、菜单间隔符的修改
  • 绝不允许擅自修改交互，菜单项图标，添加/删除菜单项!
  • 个性化样式示例： 必须以standard-v4.css为基础，重写相关样式。即先引入standard-v4.css，然后新建css，重写之。

    引入示例：

    <head>
    ...
    <link href="http://style.c.aliimg.com/sys/css/universal/alibar/standard-v4-min.css" rel="stylesheet"/>
    <link href="http://style.c.aliimg.com/XXXX/myalibar.css" rel="stylesheet"/>
    ...
</head>
